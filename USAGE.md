# TalkTranscript — 通話文字起こしツール

マイクとスピーカーの音声をリアルタイムに文字起こしし、テキストファイルに記録するツールです。
通話・会議・打ち合わせの内容を自動で書き起こします。

---

## はじめに

`TalkTranscript.exe` をダブルクリック、またはコマンドプロンプト/PowerShell から実行してください。

```
TalkTranscript.exe
```

初回起動時にマイクとスピーカーの選択画面が表示されます。
お使いの環境の GPU / CPU / メモリを自動検出し、最適な認識エンジンを選択します。

> **音声認識モデルについて:**
> 初回実行時に認識エンジンのモデルファイルが自動でダウンロードされます。
> ダウンロード先: `%LOCALAPPDATA%\TalkTranscript\Models\`

---

## 認識エンジン

| エンジン | 特徴 | モデルサイズ |
|---------|------|------------|
| **Vosk** | 軽量・リアルタイム認識 | ~48MB |
| **Whisper tiny** | 高速・準リアルタイム | ~39MB |
| **Whisper base** | バランス型 (推奨) | ~142MB |
| **Whisper small** | 高精度 | ~466MB |
| **Whisper medium** | さらに高精度 (GPU 推奨) | ~1.5GB |
| **Whisper large** | 最高精度 (GPU 推奨) | ~3.1GB |
| **SAPI** | Windows 標準 (追加ダウンロード不要) | — |

エンジンは設定メニューまたはコマンドライン引数で変更できます。

---

## 録音中の操作

| キー | 操作 |
|------|------|
| **Ctrl+Q** | 録音を停止して終了 |
| **F2** | 設定メニューを開く |
| **Ctrl+B** | ブックマークを挿入 (重要箇所のマーク) |

録音中は画面にリアルタイムで文字起こし結果と音量メーターが表示されます。

---

## 設定メニュー

`F2` キー (録音中) または起動時に `--config` オプションで設定メニューを開けます。

```
TalkTranscript.exe --config
```

変更できる項目:

| # | 項目 | 説明 |
|---|------|------|
| 1 | エンジン | 認識エンジンの切り替え |
| 2 | GPU (CUDA) | GPU アクセラレーションの有効/無効 |
| 3 | 言語 | 認識言語 (日本語/英語/自動 など) |
| 4 | 出力ディレクトリ | 文字起こしファイルの保存先 |
| 5 | 出力フォーマット | SRT / JSON / Markdown の追加出力 |
| 6 | デバイス | マイク/スピーカーの選択 |
| 7 | 録音保存 | 音声を WAV ファイルとして保存 |
| 8 | リソース制御 | CPU スレッド数/プロセス優先度 |

設定は自動的に保存され、次回起動時に引き継がれます。

---

## 出力ファイル

文字起こし結果は `Transcripts` フォルダ (または設定で指定したフォルダ) に保存されます。

- **テキスト (.txt)** — 常に出力されます
- **SRT (.srt)** — 字幕形式 (動画編集ソフトで使用可能)
- **JSON (.json)** — プログラムからの再利用・分析向け
- **Markdown (.md)** — ドキュメント共有向け

### 出力例

```
通話記録 - 2026/02/28 14:30
============================================================

[14:30:05] 自分: お疲れ様です。
[14:30:08] 相手: お疲れ様です。本日の議題ですが...

  ★ [14:35:12] ブックマーク: ブックマーク

[14:35:15] 自分: 了解しました。

============================================================
通話時間: 00:10:23  |  自分: 15件  |  相手: 12件
```

## コマンドライン引数

| 引数 | 説明 | 例 |
|------|------|----|
| `--engine <名前>` | エンジンを指定 | `--engine whisper-base` |
| `--cpu` | GPU を使わず CPU で動作 | `--cpu` |
| `--lang <コード>` | 認識言語を指定 | `--lang en` |
| `--config` | 設定メニューを開く | |
| `--whisper-only [サイズ]` | Whisper モデルの事前ダウンロード | `--whisper-only large` |
| `--diag` | 診断モード (環境チェック) | |

### 使用例

```
rem 日本語 (デフォルト) で起動
TalkTranscript.exe

rem Whisper base エンジンを GPU で使用
TalkTranscript.exe --engine whisper-base

rem CPU モードで起動
TalkTranscript.exe --engine whisper-base --cpu

rem 英語認識で起動
TalkTranscript.exe --lang en

rem Whisper large モデルを事前ダウンロード
TalkTranscript.exe --whisper-only large
```

## GPU アクセラレーション (任意)

NVIDIA GPU をお持ちの場合、CUDA を使って Whisper の認識速度を大幅に向上できます。

### 必要なもの

- NVIDIA GPU (GeForce / RTX シリーズ)
- [CUDA Toolkit 13](https://developer.nvidia.com/cuda-downloads)

CUDA Toolkit をインストールすると自動的に GPU モードが有効になります。
CUDA が見つからない場合は CPU モードで動作します (特別な設定は不要です)。

## トラブルシューティング

### マイクの音声が認識されない

1. `--diag` オプションで診断モードを実行してください:
   ```
   TalkTranscript.exe --diag
   ```
2. Windows の「設定 → プライバシー → マイク」でアプリのマイクアクセスが許可されていることを確認してください
3. マイクがミュートになっていないか確認してください

### 認識精度が低い

- より高精度なエンジンに変更してください (例: Whisper base → small)
- マイクの音量が小さい場合は、Windows のサウンド設定でマイクのゲインを上げてください
- 雑音の多い環境では、ヘッドセットマイクの使用をお勧めします

### 動作が重い

- 設定メニューの「リソース制御」から CPU スレッド数を減らすか、プロセス優先度を下げてください
- 軽量なエンジン (Vosk / Whisper tiny) に切り替えてください
- GPU が使用可能な場合は GPU モードを有効にしてください

## データの保存先

| データ | パス |
|--------|------|
| 設定ファイル | `%APPDATA%\TalkTranscript\settings.json` |
| 認識モデル | `%LOCALAPPDATA%\TalkTranscript\Models\` |
| 文字起こし結果 | `Transcripts\` (既定) または設定で指定したフォルダ |
